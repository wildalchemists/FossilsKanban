Team sync

Firebase Firestore enables real-time sync across your team. Setup takes ~10 minutes.

  1. Go to console.firebase.google.com and create a new project (free; no credit card needed).
  2. Inside the project, click Build → Firestore Database → Create database → Start in test mode (any region).
  3. Click the gear icon ⚙ → Project settings → scroll to Your apps → click </> (Web app icon) → register the app (any name).
  4. You'll see a firebaseConfig object with your keys. Copy the whole block.
  5. Open this HTML file in a text editor and find this near the top of the <script> section: 
    const SYNC_CONFIG = {
  enabled: false,
  firebase: { apiKey: "...", ... },
  boardId: 'default-board'
};
  6. Set enabled: true, paste the firebaseConfig values, and choose a boardId (e.g. 'hot-potato'). The boardId determines which document your team shares — anyone with the same file gets the same board.
  7. Save the file, push to GitHub, share the URL. Done.

⚠ Note on access: in "test mode" anyone with the URL and Firebase project ID can read/write. That's fine for a private team URL but not for a public link. To lock down later, configure Firestore Security Rules in the Firebase console.

🔓 Free tier limits: 50k reads/day, 20k writes/day, 1 GB stored. A 10-person team typing all day uses ~1-2% of any of these.
